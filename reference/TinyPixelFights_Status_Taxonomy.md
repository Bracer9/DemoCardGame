# Tiny Pixel Fights — Status Taxonomy / 状态分类法

更新时间：2026-07-03

本文定义当前战斗状态的“规则词库”。目的不是增加更多状态，而是限制状态膨胀：以后新增 Trait、Role Action、普通兵、遗物、天气或战场规则时，优先复用这里的通用状态；只有确实承担角色身份或核心循环时，才允许保留特殊状态。

相关文档：

- `reference/TinyPixelFights_Status_Display_Groups.md`：UI 如何把多个 status instance 聚合显示。
- `reference/TinyPixelFights_Role_Action_Table.md`：当前已实装 Trait / Role Action 与状态来源。
- `reference/TinyPixelFights_Trait_RoleAction_Status_Unification_Draft.md`：Trait / Role Action 状态整合与倍率化草案。

## 1. 基本原则

### 1.1 规则词库不是卡面图标数量

通用状态可以有多个，但卡面右侧不应该逐个铺开。  
卡面右侧显示的是状态组，例如“攻击变化”“防御变化”“行动限制”“持续伤害”“光环”，具体来源放在 inspector 详情中。

### 1.2 默认复用，谨慎新增

新增状态前先问：

1. 它能不能归入已有通用状态？
2. 它是不是只是数值、持续时间或来源不同？
3. 玩家是否必须把它当作一个独立规则来记？
4. 它是否需要独立图标、独立语音、独立预测说明？

如果答案不明确，默认不要新增特殊状态。

### 1.3 特殊名可以存在，但要挂靠通用名

例如 `pact / 黑暗契约` 可以保留角色味道，但 UI 与规则认知上仍然应归入“强攻”。  
玩家先理解“强攻”，再在详情中看到来源是“黑暗契约”。

### 1.4 通用名优先使用两个字

通用状态名要像玩家之间能直接说出口的战斗词汇，而不是工程描述。

优先：

- 强攻、魔涌、坚守、护咒、蓄势、再动
- 脆弱、力竭、磨损、麻痹、胆怯、燃烧

避免：

- 攻击强化、魔法攻击强化、物防上升、无法反击、受到伤害增加

## 2. 状态层级

| 层级 | 用途 | 示例 |
| --- | --- | --- |
| 通用状态 | 玩家需要反复理解的基础规则 | 燃烧、力竭、胆怯、坚守 |
| 特殊状态 | 角色身份、核心循环或强主题状态 | 播种 / 丰收、野兽之怒、黑暗契约 |
| 光环 / 场上规则 | 来源存在时持续影响全队或战场，不是单体可处理 buff | 公主祝福、占卜师预见、骑士守护 |
| 系统状态 | 只服务流程、冷却、奖励、内部判断，通常不显示为角色 buff | Role Action CD、奖励永久加成、共享盾状态 |

## 3. 当前通用状态词库

先控制在少数“玩家真的会反复遇到”的状态。以下是当前版本建议承认的通用状态。

### 3.0 推荐通用命名总览

| 大类 | 通用名 | 玩家直觉 | 备注 |
| --- | --- | --- | --- |
| Buff | 强攻 | 打得更痛 | 物理 / 普通攻击侧的增强优先归入这里。 |
| Buff | 魔涌 | 魔法变强 | 魔法伤害、燃烧增强、魔法攻击成长优先归入这里。 |
| Buff | 坚守 | 物理更硬 | 物理防御上升。 |
| Buff | 护咒 | 魔法更硬 | 魔法防御上升。 |
| Buff | 蓄势 | 下回合或稍后爆发 | 延迟变强类状态。 |
| Buff | 再动 | 还能再行动 / 再攻击 | 当前主要对应再次主动攻击机会。 |
| Debuff | 脆弱 | 被打更痛 | 受到伤害增加类状态。 |
| Debuff | 力竭 | 物理输出变弱 | 之后可定义为物理伤害减半等。 |
| Debuff | 磨损 | 魔法输出变弱 | 之后可定义为魔法伤害减半等。 |
| Debuff | 麻痹 | 不能行动 | 比“封攻”更重，当前预留。 |
| Debuff | 胆怯 | 不敢反击 | 无法反击。 |
| Debuff | 燃烧 | 持续掉血 | 当前已实装，可叠层。 |

### 3.1 持续伤害

| 通用状态 | 当前 Status ID | 类型 | 默认规则 | 当前来源 |
| --- | --- | --- | --- | --- |
| 燃烧 | `burning` | Debuff | 目标回合开始时消耗 1 层，受到 1 点魔法伤害；可叠层。 | 魔法师 Trait、魔法师 `searing-brand` |

暂不加入“中毒”。等普通兵、遗物或天气中真的出现稳定 DoT 体系时再加入，避免提前膨胀。

### 3.2 攻击变化

| 通用状态 | 当前 Status ID | 类型 | 默认规则 | 当前来源 |
| --- | --- | --- | --- | --- |
| 强攻 | `harvest`, `pact`, `reward-attack` | Buff | 提高主动攻击或基础攻击。具体数值和持续时间看来源。 | 农民、怪物、奖励 |
| 魔涌 | `magic-power`, `charged`, `reward-magical-attack` | Buff / Aura | 提高魔法伤害或魔法系攻击。 | 占卜师、魔法师、奖励 |
| 力竭 | `weakness`, `weakness-pending` | Debuff | 降低主动攻击输出。后续若拆分物理 / 魔法，可让力竭专指物理输出下降。 | 德鲁伊 Trait、德鲁伊 Role Action |
| 磨损 | 暂无统一 Status ID | Debuff | 预留为魔法输出下降。当前还没有稳定来源。 | 未来魔法系 debuff |

说明：

- `magic-power` 当前在显示上作为 Aura 折叠，但规则认知仍归入“魔涌”。
- `harvest` 可以在 UI 保留“丰收”特殊名，但它的数值本质是“强攻”。

### 3.3 防御变化

| 通用状态 | 当前 Status ID | 类型 | 默认规则 | 当前来源 |
| --- | --- | --- | --- | --- |
| 坚守 | `guarded`, `supply-guard`, `reward-physical-defense` | Buff | 物理防御上升。具体持续时间看来源。 | 骑士、农民、奖励 |
| 护咒 | `reward-magical-defense` | Buff | 魔法防御上升。 | 奖励 |
| 破防 | `rage` | Buff with drawback | 物防 / 魔防下降。 | 狂战士 `war-cry` |

说明：

- `rage` 同时包含正面和负面效果。显示时可以拆到“行动变化”和“破防”两个组里，详情仍写来源是狂战士战吼。

### 3.4 行动限制 / 行动变化

| 通用状态 | 当前 Status ID | 类型 | 默认规则 | 当前来源 |
| --- | --- | --- | --- | --- |
| 封攻 | `attack-sealed` | Debuff | 本回合不能主动攻击。 | 魔法师 `arcane-channel` |
| 胆怯 | `challenged` | Debuff | 本回合不能反击。 | 狂战士 `challenge` |
| 麻痹 | 暂无统一 Status ID | Debuff | 预留为“无法行动”。是否包含 Role Action 需要另行定义。 | 未来控制系效果 |
| 再动 | 无统一 Status ID，当前由角色行动次数字段处理 | System / Buff-like | 允许角色本回合再次主动攻击。 | 占卜师 `star-reading`、狂战士 `war-cry`、农民 `field-work` |

说明：

- “再动”目前不是统一 status，而是行动次数系统。它仍然是玩家认知上的通用状态，未来可以考虑给 DTO / UI 一个统一 display group。
- “封攻”和“麻痹”不要混用：封攻只禁止主动攻击；麻痹才是更重的无法行动。

### 3.5 脆弱 / 标记 / 触发伤害

| 通用状态 | 当前 Status ID | 类型 | 默认规则 | 当前来源 |
| --- | --- | --- | --- | --- |
| 脆弱 | `searing-brand` | Debuff | 受到特定类型伤害时增加伤害。当前为魔法伤害 +1。 | 魔法师 `searing-brand` |
| 命印 | `marked` | Debuff | 下一次攻击我方目标时，50% 伤害减半，50% 伤害 +1。 | 占卜师 `fate-mark` |
| 猎印 | `prey` | Debuff | 本回合每次受到 0 点 HP 伤害时，追加绝对伤害。 | 怪物 `predatory-gaze` |

说明：

- `marked` 和 `prey` 现在仍偏特殊，因为规则不只是简单加减数值。
- 如果未来出现大量“标记”效果，应拆成更明确的二字名，例如“命印”“猎印”“破绽”，而不是所有都叫 mark。

### 3.6 延迟状态

| 通用状态 | 当前 Status ID | 类型 | 默认规则 | 当前来源 |
| --- | --- | --- | --- | --- |
| 蓄势 | `charged-pending` | Buff | 下个己方回合开始转为 `charged`。 | 魔法师 `arcane-channel` |
| 播种 | `harvest-pending` | Buff | 下个己方回合开始转为 `harvest`。 | 农民 Trait / `field-work` |

说明：

- 延迟状态容易让玩家迷惑。只有当“等一回合后变强”本身是机制重点时才显示。
- 大量延迟效果出现后，应合并为“延迟效果”显示组，详情列来源。

## 4. 当前特殊状态

特殊状态允许保留独立名称，但数量必须少。它们通常属于角色核心玩法。

| 特殊状态 | Status ID | 类型 | 来源 | 保留理由 | 是否可驱散 |
| --- | --- | --- | --- | --- | --- |
| 播种 | `harvest-pending` | Buff | 农民 | 农民核心节奏的一半。 | 是 |
| 丰收 | `harvest` | Buff | 农民 | 农民核心节奏的一半。 | 是 |
| 野兽之怒 | `beast-rage` | Buff | 怪物 Trait | 怪物因公主死亡获得的身份变化，影响基础攻击。 | 否 |
| 狂怒 | `rage` | Buff / Drawback | 狂战士 `war-cry` | 高风险高收益的狂战士身份表达。 | 是 |
| 黑暗契约 | `pact` | Buff | 怪物 `dark-pact` | 失血换爆发，主题强且是一次性爆发。UI 可归入“强攻”。 | 是 |
| 命印 | `marked` | Debuff | 占卜师 `fate-mark` | 概率类强预测影响，暂时保留特殊。 | 是 |
| 猎印 | `prey` | Debuff | 怪物 `predatory-gaze` | “0 伤害也会被惩罚”的特殊规则。 | 是 |

新增特殊状态必须满足至少一个条件：

- 是某个角色的主循环。
- 会持续影响多个决策，而不是只改一次数值。
- 不能用现有通用状态表达清楚。

## 5. 光环 / 场上规则

这些效果不应当被当作普通 buff/debuff 刷满角色卡右侧。它们应折叠进 Aura 组，或者由独立 HUD / 预测说明表达。

| Aura / 场上规则 | 当前 Status ID / 表示 | 来源 | 规则性质 | 是否可驱散 |
| --- | --- | --- | --- | --- |
| 公主祝福 | `blessing` | 公主 Trait | 公主存活时，我方回合开始全队治疗。 | 否 |
| 预见 | `foresight` | 占卜师 Trait | 占卜师存活时，全队概率减伤。 | 否 |
| 星读魔力 | `magic-power` | 占卜师 Trait | 占卜师存活时，全队魔法伤害与燃烧增强。 | 否 |
| 骑士守护 | `guard` | 骑士 Trait | 骑士存活时，全队共享一次主动物理代伤。 | 否 |
| 共享盾 | 非角色 status | 防御阵形 / 举盾号令 | 战场中间的队伍护盾，有自己的盾量、物防、魔防。 | 不按驱散处理 |

一句话原则：

> 光环是“场上规则”，不是“贴在角色身上的可驱散状态”。

以后所有光环默认：

- 不算普通 buff。
- 不可被德鲁伊等“移除 buff”类效果驱散。
- 卡面右侧不逐个显示，只折叠到 Aura 组。

## 6. 系统 / 内部状态

以下状态或效果不应该优先作为普通 buff/debuff 显示。

| 状态 | 当前表示 | 说明 |
| --- | --- | --- |
| 共享盾数值 | Player shared shield fields | 应在盾 HUD 显示，不显示成角色 buff。 |
| Role Action 冷却 | Role Action cooldown fields | 显示在 Role Action 按钮上，不作为角色 buff。 |
| 奖励永久成长 | `reward-attack`, `reward-magical-attack`, `reward-physical-defense`, `reward-magical-defense` | 现在用 status 实现，但设计上更像本局遗物 / 永久成长。UI 可并入数值变化详情，不必作为专属 buff 图标。 |
| 本回合攻击次数 | Character action counters | 影响可行动状态，应由卡片可行动表现和详情说明，不一定显示成 buff。 |

## 8. 暂不加入的通用状态

这些状态可以留作未来扩展，但当前不要为了“像卡牌游戏”而提前加入。

| 候选状态 | 暂不加入原因 | 未来加入条件 |
| --- | --- | --- |
| 中毒 | 当前没有稳定毒系角色 / 普通兵 / 天气。 | 出现多来源持续固定伤害体系。 |
| 麻痹 | 当前只有“封攻”，还没有完整无法行动体系。 | 需要区别“不能攻击”“不能 Role Action”“Cost 增加”时再定义。 |
| 睡眠 / 眩晕 | 控制过强，容易破坏节奏。 | 单人爬塔或普通兵体系需要硬控时再讨论。 |
| 沉默 | 当前 Trait / Role Action 结构还在扩展中，过早引入会复杂化 UI 与预测。 | Role Action 成为主要战斗资源后再考虑。 |
| 流血 | 与燃烧 / 中毒容易重复。 | 需要物理 DoT 体系时再加入。 |

## 9. 新增状态流程

新增状态时按这个顺序处理：

1. 先选择分类：通用、特殊、光环、系统。
2. 如果是通用状态，优先复用已有 ID / display group。
3. 如果必须新增特殊状态，在本文件“特殊状态”表中说明保留理由。
4. 明确是否可驱散 / 可净化：默认可，只有不可时写明。
5. 明确是否叠层、是否消耗、何时过期。
6. 更新：
   - `reference/TinyPixelFights_Role_Action_Table.md`
   - `reference/TinyPixelFights_Status_Display_Groups.md`
   - 双语 locale
   - 如果影响攻击结果，更新预测 HUD。

## 10. 当前结论

当前版本不需要再增加大量通用状态。  
最优先做的是把已有状态按上面的 taxonomy 归类，并让 UI 聚合同类项：

- 卡面右侧显示状态组，不显示每个来源。
- inspector 中显示组详情和来源。
- 光环不算普通 buff，不可驱散，统一折叠。
- 奖励永久成长和共享盾状态不应占用角色 buff/debuff 空间。

这样可以保留角色个性，同时避免后续 8 个英雄、普通兵、副官、遗物和天气全部堆成“状态地狱”。
