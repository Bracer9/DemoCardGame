# Tiny Pixel Fights - Build 成型与生存节奏方案

更新时间：2026-07-06

本文讨论 testplay 暴露的问题：当前游戏鼓励队伍 build，但角色常在 build 成型前被集火击败；3 turn 补员使体验变成“打掉、摇人、再打掉”，成长内容难以兑现。本文只讨论设计方案，不代表实装决定。

## 1. 当前机制确认

### 1.1 战斗与行动

- 每个玩家己方 turn 有 5 AP。
- 主动攻击消耗角色 Cost。
- Role Action 默认不消耗主动攻击权，具体是否封印攻击或追加攻击次数由效果决定。
- 当前主要输出来源包括普通攻击、反击、Trait 伤害、燃烧、绝对伤害、Role Action 造成的状态放大。
- 部分效果直接改变行动经济：`royal-command`、`star-reading`、`war-cry`、`field-work`。

### 1.2 共享盾

- 共享盾是队伍盾池，不属于单个角色。
- 公共防御指令：首次 2 AP 获得共享盾 2；同 turn 若盾仍存在，可追加 1 AP 使当前盾值 +2。
- 共享盾吸收主动攻击、反击、Trait / 技能、状态造成的物理 / 魔法伤害。
- 共享盾不吸收绝对伤害。
- 共享盾在伤害管线中先于骑士守护结算。
- 未消耗的盾在下个己方回合开始前到期。

### 1.3 Trait / Role Action 状态

- 通用状态包括强攻、猛击、魔涌、咏唱、坚守、护咒、脆弱、空虚、力竭、磨损、燃烧、战栗。
- 通用状态默认可驱散 / 可净化。
- 特殊状态包括公主治疗光环、星读加护、播种 / 丰收、野兽之怒、命运标记、猎物、黑暗契约、守护誓约等。

### 1.4 士兵成长

- 开局计划为每名玩家选择 1 名英雄 + 2 名士兵。
- 士兵 Rank0 有初始 Trait，无 Role Action。
- Rank1 获得小幅属性成长、Trait 强化和持续队伍光环。
- Rank2 最大 HP +5，升阶瞬间 HP 全回复，解锁唯一 Role Action。
- Rank2 后可作为副官，Rank1 光环继续生效。

## 2. 问题诊断

### 2.1 核心矛盾

当前不是“没有防御”，而是：

> 击杀一个关键角色带来的收益太直接，成长投资兑现太慢，补员机制又把局面推向反复换人。

玩家的理性选择会变成：

1. 集火对方核心英雄或士兵。
2. 在对方升级 / build 成型前移除关键 Trait / Role Action。
3. 等补员后继续重复。

这样会削弱三个体验：

- 英雄成长：Lv1 / Lv2 / Lv3 选择来不及体现。
- 士兵成长：Rank1 光环和 Rank2 Role Action 难以稳定进入局面。
- Build 快感：玩家不是在组体系，而是在抢先拆体系。

### 2.2 为什么 3 turn 补员不能解决

补员解决的是“场上没人可玩”，没有解决“成长投资死亡后断裂”。

如果补员默认是新单位：

- 玩家会觉得队伍一直在重置。
- 死亡不再是一个阶段性损失，而是 build 线被截断。
- 对局变成“消耗对方当前棋子”，而不是“围绕核心路线成长”。

### 2.3 当前高危因素

| 因素 | 风险 |
|---|---|
| 5 AP 行动空间 | 允许同 turn 内多次集中攻击同一目标。 |
| Role Action 不默认消耗攻击权 | 一些角色能同时铺状态、放大伤害、再攻击。 |
| `war-cry` / `star-reading` / `royal-command` | 直接改变行动次数或 AP 节奏，容易制造爆发 turn。 |
| 绝对伤害 | 绕过共享盾和防御，容易成为终结低 HP 单位的工具。 |
| 士兵 Rank1 光环强 | 一旦士兵活下来很有价值，因此对手更想提前集火士兵。 |
| 英雄升级投资大 | 若英雄在 Rank3 前阵亡，进阶 class 内容无法成为对局核心。 |

## 3. 设计目标

解决方案不应只是“血量变厚”。更合理的目标是：

1. 角色有机会活到第一次成长兑现。
2. 成长投资不因一次死亡完全归零。
3. 集火仍然是可选战术，但不应永远是默认最优解。
4. 补员应该服务 build，而不是替代 build。
5. 对局不能拖成纯治疗 / 盾循环。

## 4. 方案组 A：提前兑现成长

这组方案不直接削弱击杀，而是让玩家更早拿到 build 的“味道”。

### A1. Round 1 固定第一次英雄升级

规则：

- Round 1 双方进入奖励阶段。
- 第一次奖励固定包含“英雄职业训练”。
- 该奖励可以低价或免费，让开局英雄立刻解锁 1 个 Basic Role Action。

优点：

- 玩家很快进入“角色不只会攻击”的状态。
- 核心英雄即使之后阵亡，也至少展示过路线。
- 对战模式更适合作为 Role Action prototype。

风险：

- 一开始认知负担增加。
- `royal-command`、`star-reading` 等行动经济按钮会更早进入环境。

适合测试：

- 当前已经有 8 英雄两选一 Role Action，应该尽快让玩家实际体验。

### A2. 第一次士兵成长提前

规则：

- 初始两名士兵中，玩家可以在开局后立刻选择 1 名获得 Rank1。
- 或 Round 1 奖励固定给“士兵训练素材”。

优点：

- 士兵初期不只是可牺牲单位。
- Rank1 光环可以马上让队伍有构筑方向。

风险：

- Rank1 士兵会变成第一集火目标。
- 士兵光环需要确保不会强到压过英雄。

### A3. 升级奖励附带立即治疗

规则：

- 任意英雄升级时，回复该角色最大 HP 的 50%。
- 任意士兵 Rank1 时，回复 3 HP。
- 士兵 Rank2 继续保持当前文档中的最大 HP +5 和满血。

优点：

- 成长本身就是防止被立刻打掉的窗口。
- 玩家购买升级会有即时战术价值。

风险：

- 若升级奖励出现太频繁，会形成“靠升级治疗”的拖局。

建议：

- 第一版可以只给英雄升级回血，士兵保留 Rank2 满血。

## 5. 方案组 B：让核心角色不那么容易被秒

这组方案直接降低早期集火收益。

### B1. 起始英雄获得“指挥保护”

规则：

- 开局选择的英雄拥有一次性保护。
- 该英雄第一次受到致命伤害时，HP 保留为 1，并移除该保护。
- 保护在该英雄完成 Lv1 升级后也可以自动移除，二选一。

优点：

- 保证开局英雄至少有一个展示窗口。
- 规则直观，UI 可用一个小盾章显示。

风险：

- 可能让玩家觉得“第一轮打英雄很亏”，转而无脑打士兵。

变体：

- 保护只在 Round 1-2 生效。
- 保护只防主动攻击，不防燃烧 / 绝对伤害。

### B2. 英雄专属“重伤”而非直接退场

规则：

- 英雄 HP 到 0 时不立刻离场，而是进入重伤。
- 重伤英雄不能攻击，不能使用 Role Action，Trait 暂停。
- 若己方下个 turn 结束前没有被治疗到 1 HP 以上，则退场。

优点：

- 给对手“击倒”奖励，也给拥有者“救援”选择。
- 治疗 / 净化 / 盾保护有了更清楚的救援用途。
- 不直接增加 HP，但增加 build 核心的可挽回性。

风险：

- UI 和规则复杂度上升。
- 需要处理重伤时是否占场、是否可被继续攻击。

推荐口径：

- 第一版只对英雄生效，不对士兵生效。

### B3. 单 turn 集火衰减

规则：

- 同一目标每个 turn 第一次受到主动攻击正常结算。
- 第二次及之后受到主动攻击时，目标自动获得本次伤害前的 1 turn 坚守 / 护咒，按伤害类型应用。

优点：

- 不禁止集火，但让连续打同一个目标收益下降。
- 鼓励分散攻击、铺状态、先破盾再寻找窗口。

风险：

- 需要额外 UI 说明，否则玩家会疑惑伤害为什么变低。
- 可能与共享盾、防御、骑士守护叠得过厚。

轻量变体：

- 只对英雄生效。
- 只在 Round 1-4 生效。

### B4. 早期英雄伤害上限

规则：

- Round 1-3，单个英雄每个敌方 turn 最多损失其最大 HP 的 50%。
- 超出的非绝对伤害被截断。

优点：

- 强行保证英雄不会在 build 前蒸发。
- 最适合 prototype 阶段验证成长系统。

风险：

- 有“规则保护感”，不够自然。
- 可能削弱爆发型角色的成就感。

建议：

- 只作为短期测试辅助，不建议长期正式化。

## 6. 方案组 C：让死亡不摧毁成长投资

这组方案比单纯加血更接近长期单机 / JRPG 养成方向。

### C1. 死亡后保留 Rank / Lv

规则：

- 被击败的英雄或士兵离场。
- 若之后通过补员 / 复活 / 奖励回到战场，保留原有成长等级。

优点：

- 玩家不会觉得升级资源白费。
- 对手击杀仍然有短期收益：该单位暂时离场。

风险：

- 若回归太快，会降低击杀意义。

建议：

- 成长保留，但回归需要等待补员窗口或支付奖励。

### C2. 补员阶段改为“复归 / 新招募 / 训练”三选一

规则：

补员窗口提供三类选项：

1. 复归：让一个已退场单位回到战场，保留成长，HP 为 50%。
2. 新招募：获得新的士兵或英雄。
3. 训练：不补人，但给现有单位升级素材。

优点：

- 补员不再只是摇人。
- 玩家可以围绕原有 build 继续经营。

风险：

- 奖励窗口信息量上升。

推荐：

- 这是最符合“build 成型”的补员改法。

### C3. 死亡转化为“遗志”

规则：

- 成长过的单位死亡后，给我方一个一次性遗志奖励。
- 英雄死亡：获得 1 个英雄训练折扣或指定类型奖励权重提高。
- 士兵死亡：同职业下一张士兵获取时自动 Rank1，或当前场上一名单位获得对应通用状态。

优点：

- 死亡仍然痛，但不会完全断线。
- 很适合未来单机叙事。

风险：

- 可能鼓励主动送死刷遗志。

限制：

- 每名单位每局只触发一次。
- 只有 Rank1 以上单位死亡才触发。

### C4. 副官继承

规则：

- Rank2 士兵死亡时，不完全消失。
- 可在下一次奖励阶段把她转为副官，附到英雄上，但不再作为场上单位行动。

优点：

- 士兵成长不会因为被打掉而完全浪费。
- 也能解释“副官”系统不是纯额外增强，而是阵亡后的角色转化。

风险：

- 叙事上需要处理“阵亡但成为副官”的语义，可以改成“撤退后担任副官”。

## 7. 方案组 D：调整补员与场面规模

### D1. 补员进入手牌 / 后备区，而不是直接上场

规则：

- 补员获得单位，但先进入后备区。
- 玩家需要花 AP 或奖励窗口操作将后备单位部署到空位。

优点：

- 补员不再立刻变成下一轮被杀目标。
- 玩家可以规划“什么时候部署核心”。

风险：

- 增加一个后备区 UI。

### D2. 开局 1 英雄 + 3 士兵

规则：

- 初始队伍从 3 人变为 4 人。

优点：

- 场面更宽，单个核心被打掉后仍有体系残留。
- Shield / Aura / Role Action 的范围价值更明显。

风险：

- 进攻火力也上升，可能反而更快死人。
- 初始认知负担变高。

建议：

- 只有在引入焦点衰减或早期保护后再测试。

### D3. 每方固定 1 个“后备士兵槽”

规则：

- 开局仍是 1 英雄 + 2 士兵上场。
- 另有 1 个后备士兵不可攻击，不提供 Trait。
- 有单位退场时，玩家可选择是否把后备士兵部署。

优点：

- 降低场面清空感。
- 不直接增加开局认知负担。

风险：

- 需要后备 UI。

## 8. 方案组 E：改变击杀收益与攻击目标选择

### E1. 击杀不直接奖励 BP，破盾 / Role Action / 生存奖励更重要

规则：

- BP 不因击杀大量获得。
- BP 低保、破盾、主动攻击造成伤害、使用 Role Action 等仍存在。
- 可以加入“回合结束时场上有 Rank1+ 单位存活，获得 1 BP”的成长倾向奖励。

优点：

- 玩家不再只为了资源最大化而集火。
- 鼓励保护成长单位。

风险：

- 防守队可能滚 BP。

限制：

- 每 turn 最多 1 BP。
- 只检查一名最高 Rank 单位即可，不按数量叠加。

### E2. 连续攻击同一目标的 BP 收益递减

规则：

- 每个 turn 第一次攻击某目标造成伤害可获得 BP。
- 第二次及之后攻击同一目标不再因造成伤害获得 BP。

优点：

- 不改伤害，也能降低集火的资源收益。
- 玩家仍可选择集火，但牺牲成长资源。

风险：

- 对局短期结果仍可能由集火决定。

### E3. 打 Rank 高单位不等于直接赚

规则：

- 击败 Rank1+ 士兵或 Lv1+ 英雄时，不额外给攻击方奖励。
- 相反，被击败方获得一个小的补偿，例如下一次相关升级折扣。

优点：

- 减少“先杀培养对象就是最优”的压迫感。

风险：

- 可能让玩家觉得击杀高价值目标没奖励。

折中：

- 攻击方获得短期 BP，被击败方获得长期折扣。

## 9. 方案组 F：调整共享盾与公共防御

### F1. 共享盾保留到敌方 turn 结束

当前：

- 未消耗盾在下个己方回合开始前到期。

方案：

- 共享盾在经过一次完整敌方 turn 后才到期。

优点：

- 玩家更愿意提前防御。
- 防御不容易因为顺序或小伤害浪费。

风险：

- 可能明显拖局。

建议：

- 若采用，需要降低盾值或限制连续使用。

### F2. 公共防御附带“反集火保护”

规则：

- 使用公共防御后，除共享盾外，我方最低 HP 角色获得 1 turn 坚守。

优点：

- 公共防御更能保护核心。
- 与 Shieldmaiden 形成清晰协同。

风险：

- 与 Shieldmaiden / Knight / Princess 叠加可能过厚。

### F3. 盾被击破时不再让被攻击角色承受全部后续爆发

规则：

- 若某次攻击击破共享盾，溢出到 HP 的伤害最多为本次原始伤害的 50%。

优点：

- 盾不再只是被第一击清掉。
- 反集火效果强。

风险：

- 规则不够直观。
- Monster / Barbarian 破盾设计需要重测。

## 10. 方案组 G：限制爆发来源

### G1. `star-reading` 加冷却或禁止蓄力目标

问题：

- 再行动是最强行动经济之一。

方案：

- `star-reading` 冷却 1 turn。
- 或目标若拥有咏唱，不能被 `star-reading` 选中。

优点：

- 降低魔法炮台一回合秒人的概率。

风险：

- Oracle 的主动价值下降。

### G2. `war-cry` 的破盾绝对伤害延后

问题：

- `war-cry` 兼具额外攻击、防御代价、破盾绝对伤害，爆发密度高。

方案：

- 击破共享盾时，不立即造成绝对伤害，而是给目标 1 turn 脆弱。
- 或绝对伤害只在目标没有共享盾保护时生效。

优点：

- 保留狂战士破阵身份，降低一回合杀穿。

风险：

- 狂战士爽感下降。

### G3. 绝对伤害早期上限

规则：

- Round 1-3，单次绝对伤害对英雄最多造成 3 点，对士兵最多造成 2 点。

优点：

- 防止 Monster / Duelist 等绕盾过早击杀核心。

风险：

- 绝对伤害的规则例外变多。

建议：

- 若不用重伤系统，可作为过渡。

## 11. 方案组 H：奖励节奏改造

### H1. 奖励阶段提供“保护型奖励”

奖励池加入：

- 选择一名英雄，获得一次致命保护。
- 选择一名士兵，立即 Rank1 并回复 50% HP。
- 全队获得共享盾 2，持续到下个己方 turn。
- 选择一名已退场单位，复归到战场，HP 50%。

优点：

- 玩家可以根据局势选择保核心，而不是只拿输出成长。

风险：

- 奖励池变复杂。

### H2. 每次英雄升级强制附带“成长护航”

规则：

- 英雄 Lv1 / Lv2 / Lv3 后，获得 1 turn 坚守与护咒。
- 或获得一次性致命保护，持续到下个己方 turn 结束。

优点：

- 保证刚升级的英雄不被立刻拆掉。
- 玩家能看见升级后的 Role Action / Trait 至少用一次。

风险：

- 升级窗口会成为防御窗口，可能拖慢节奏。

### H3. Rank3 进阶 class 保底窗口

规则：

- 当英雄达到 Lv3 进阶后，立即满血或回复 50% HP。
- 并获得一次“进阶首秀”：下个己方 turn 前不可被致命击败，最低保留 1 HP。

优点：

- 确保进阶 class 内容能被实际观察。
- 很适合 prototype 阶段验证高级 Role Action。

风险：

- 进阶瞬间过强，需要控制出现频率。

## 12. 方案组 I：改变胜利与局内目标

如果胜利条件只有“杀光对面”，玩家自然会尽快杀核心。可以加入轻量目标，让 build 有时间展开。

### I1. 战术点 / 战线优势

规则：

- 每隔 2 round 结算一次战线优势。
- 如果我方场上有更多存活单位，或有 Rank1+ 单位存活，获得 BP / 奖励权重。

优点：

- 保护单位本身变成目标。
- 与成长系统自然相连。

风险：

- 可能鼓励龟缩。

### I2. 试验场模式临时胜利目标

规则：

- 对战 prototype 不一定必须以全灭为唯一胜利。
- 可以测试“先完成 3 次成长选择”或“拥有 2 名 Rank2 / Lv2 单位时获得优势奖励”。

优点：

- 更符合当前对战模式作为单机战斗系统试验场的定位。

风险：

- 与传统对战直觉不同。

## 13. 推荐测试包

### Package 1：最小改动，优先推荐

目标：不大改 UI，不新增重伤系统，先确认 build 是否能成型。

内容：

1. Round 1 固定出现第一次英雄职业训练。
2. 英雄每次升级回复 50% HP。
3. 士兵 Rank2 继续最大 HP +5 并满血。
4. 补员阶段允许选择“复归已退场单位，保留成长，HP 50%”，而不只是新招募。

预期：

- 玩家更早体验 Role Action。
- 升级不再只是长期投资，也能救命。
- 死亡不会完全抹掉 build。

### Package 2：保护核心

目标：保证主英雄有发育窗口。

内容：

1. 开局英雄获得一次性致命保护，持续到 Round 3 或 Lv1。
2. 同一目标每 turn 第二次被主动攻击时，自动获得对应类型半伤。
3. 击杀不额外奖励 BP，连续打同一目标不重复获得伤害 BP。

预期：

- 集火仍可做，但不是唯一答案。
- 低血英雄有机会进入成长。

### Package 3：重伤系统

目标：让击倒与死亡之间出现救援玩法。

内容：

1. 英雄 HP 到 0 进入重伤，不立刻退场。
2. 重伤英雄 Trait / Role Action 暂停。
3. 己方下个 turn 结束前若被治疗到 1 HP 以上则恢复，否则退场。
4. 士兵仍直接退场，保持节奏。

预期：

- 治疗 / 盾 / 净化有明确救援意义。
- 英雄 build 不容易被一次爆发删除。

### Package 4：成长保留

目标：接受死亡频率，但不让死亡摧毁成长投资。

内容：

1. 死亡单位保留 Lv / Rank。
2. 补员窗口三选一：复归、新招募、训练。
3. Rank1+ 单位死亡后给被击败方一次对应成长折扣。

预期：

- 对局仍然激烈。
- 玩家能持续追同一个 build 方向。

## 14. 我目前的倾向

优先不要只靠全员加 HP。全员加 HP 会延长对局，但不一定解决“成长投资被删除”的心理问题。

更推荐先测试：

1. **Round 1 固定第一次英雄职业训练。**
2. **英雄升级附带回复。**
3. **补员阶段允许复归已退场且保留成长的单位。**
4. **如果仍然死太快，再给开局英雄一次性致命保护。**

这样做的核心逻辑是：

- 让 build 更早开始。
- 让升级有即时防守价值。
- 让死亡是 tempo 损失，而不是成长线归零。

## 15. Playtest 记录指标

下一轮测试建议记录：

| 指标 | 目标 |
---|---|
| 第一个英雄 Lv1 出现 round | 是否稳定在 Round 1-2。 |
| 第一个英雄 Lv3 出现 round | 是否至少有 30%-50% 对局能看到。 |
| Rank2 士兵出现次数 | 是否每局至少能看到 1 次。 |
| 单位平均存活 turn | 核心英雄是否能活过 2 个己方 turn。 |
| 补员选择 | 玩家更常选复归、新招募还是训练。 |
| 集火频率 | 同一目标单 turn 被攻击 2 次以上的频率。 |
| 防御指令使用率 | 共享盾是否成为真实选择，而非被忽视或固定最优。 |
| 对局时长 | 熟练局是否仍在可接受范围。 |

## 16. 暂不建议的方向

| 方向 | 原因 |
|---|---|
| 只把全员 HP 大幅提高 | 容易拖局，但不保证 build 投资不被删除。 |
| 只提高共享盾数值 | 可能让防御成为固定最优，且被绝对伤害绕过。 |
| 让补员更频繁 | 会强化“打掉再摇人”的循环。 |
| 给所有单位免费复活 | 击杀失去意义，节奏可能变钝。 |
| 过早加入复杂天气 / 遗物反制 | 会掩盖核心节奏问题，不利于定位。 |
