## 0. 本文和上一份的区别

上一份 `TinyPixelFights_GDD_Brainwriting_20260627.md` 做了大量整理、分类和风险评估，但确实偏保守，更多是在帮用户已有想法搭架子。

1. **外部吸收**：从 STS、炉石、影之诗、LoR、Monster Train、Balatro、Marvel Snap、Darkest Dungeon、Roguebook、Griftlands、MTG/Gwent 等设计中提炼机制语法。

本文的目标不是“立刻决定做什么”，而是让后续讨论有足够多的材料。

## 1. 外部游戏机制拆解与翻译

### 1.1 Slay the Spire：奖励选择定义路线

我从 STS 借的不是“卡组构筑”本身，而是：

- 战斗后奖励不是单纯变强，而是让玩家逐步定义 build。
- 三选一卡牌奖励让随机性带有选择权。
- 遗物提供 run 内永久被动，能改变一张卡、一类牌或整套策略的价值。
- 地图路线把风险和奖励绑在一起：普通战斗、精英、商店、营火、事件。

翻译到本项目：

- Tiny Pixel Fights 不应复制“抽牌/删牌/压缩牌组”，而应做 **队伍构筑**。
- 奖励不是“拿一张牌”，而是“升级英雄、培养普通兵、获得遗物、改变战场倾向”。
- 对战模式没有地图，但可以把“固定 round 奖励窗口”做成一场战斗内的微型路线节点。

我的机制提案：

| 提案 | 说明 |
|---|---|
| 奖励三分法 | 每次固定奖励窗口尽量给 1 个英雄成长、1 个普通兵成长、1 个战术/遗物方向，避免纯随机全是同类 |
| 队伍倾向标签 | 玩家选过的奖励会给队伍打标签，如 `royal`、`arcane`、`ironwall`、`wild`，后续奖励轻微加权 |
| 跳过奖励 | 玩家可以跳过三选一，不花 BP、不拿奖励，保留资源给下次奖励窗口 |
| 微型精英目标 | 某 round 出现挑战目标，例如“本 round 首次破盾获得额外奖励候选”，把 STS 精英风险压缩成对战目标 |

### 1.2 Hearthstone：Discover 与关键词压缩

炉石很强的一点是：随机经常被包装成“发现/三选一”，所以玩家感觉自己参与了命运，而不是被系统发牌摆布。

可借点：

- Discover 类机制：从有限候选中选择。
- 关键词压缩：复杂规则用短词和统一模板表达。
- Dungeon Run 式单人：boss 连战、被动宝藏、主动宝藏、战后奖励。
- 强反馈：卡牌出手、攻击、死亡、秘密触发都很明确。

翻译到本项目：

- BP 奖励不该是“随机得到一个东西”，而应是“发现一项成长”。
- 英雄升级、普通兵军衔、遗物都可以用 Discover 的短选择节奏。
- 技能和奖励要尽量关键词化，否则卡面与 hover 会爆炸。

我的机制提案：

| 关键词 | 可能含义 |
|---|---|
| 立功 | 该单位完成特定条件后获得 BP 或军衔进度 |
| 护送 | 保护相邻/指定英雄，触发防守奖励 |
| 破阵 | 破坏共享盾后触发 |
| 集令 | 本回合后续普通兵或英雄获得小加成 |
| 星兆 | 下一次随机判定可见或可重掷 |
| 誓约 | 获得强力效果，但绑定一个代价 |
| 传承 | 普通兵离场成为副官或遗物 |
| 觉醒 | 英雄第三次成长或转职触发 |

### 1.3 Shadowverse / Worlds Beyond：有限强成长和后手补偿

影之诗的进化点有几个启发：

- 它是有限资源，不是每回合都发生。
- 它有强仪式感，进化后的卡牌明显不同。
- 后手资源更多，改善先手节奏优势。
- Worlds Beyond 的 Super-Evolve 更像中后期关键爆发点。

翻译到本项目：

- 英雄转职不应该像普通 buff 一样悄悄加数值，应该像“进化”一样有明确节点。
- 对战中后手补偿可以不是粗暴 +AP，而是给一次“觉醒令”或 BP 预备。
- 成长系统要有次数限制，否则会变成滚雪球。

我的机制提案：

| 提案 | 说明 |
|---|---|
| 觉醒令 | 每方一局最多 1 次，让一个英雄的升级直接变成高级 Class 候选 |
| 后手战旗 | 后手开局持有，可在己方回合选择 +1 AP 或 +2 BP，一次性 |
| 迟发进化 | Round 3 后才允许第三次英雄升级，避免第一轮爆发成型 |
| 转职镜头 | 仅第三次升级触发短演出、换立绘、职业纹章，不拖每个小升级 |

### 1.4 Legends of Runeterra：公开升级条件与攻防节奏

LoR 的 Champion level-up 很值得看：

- 升级条件公开。
- 玩家可以围绕条件打。
- 对手也能看见威胁正在接近。
- 升级后卡牌改变，不只是数值加一。

翻译到本项目：

- 英雄升级不一定只来自三选一奖励，也可以有“个人成长条件”。
- BP 是全局成长资源，但每个英雄也可以有轻量个人进度。
- 玩家和对手都应该看到英雄离升级还有多远。

我的机制提案：

| 英雄 | 个人成长条件示例 |
---|---|
| 公主 | 累计有效治疗 4 次后，提高公主升级奖励出现率 |
| 农民 | 完成 2 次春播/丰收循环后，可获得一次免费属性成长候选 |
| 魔法使 | 炎上造成累计 3 点伤害后，魔法系奖励权重提高 |
| 占卜师 | 预见成功 2 次后，获得星兆类奖励候选 |
| 德鲁伊 | 成功驱散 1 次后，衰弱/控制奖励权重提高 |
| 狂战士 | 余波命中 2 次后，破阵奖励权重提高 |
| 骑士 | 守护触发 2 次后，圣盾/铁壁奖励权重提高 |
| 怪物 | 绝对追击触发 1 次后，野性/暴走奖励权重提高 |

### 1.5 Monster Train：位置、层、Champion upgrade

Monster Train 的启发：

- 它不是只靠卡牌文本，而是把战场位置做成核心系统。
- Champion upgrade 让主角单位在 run 内定型。
- 不同阵营组合形成玩法差异。

翻译到本项目：

- Tiny Pixel Fights 目前已有 slot 和邻接，但还没把站位做深。
- 高级 Class 就是我们的 Champion upgrade。
- 副官相当于“第二阵营/第二来源”给英雄注入新玩法。

我的机制提案：

| 提案 | 说明 |
|---|---|
| 前锋/后卫 | 仍是 1x4，但 slot 0/1 为前锋，slot 2/3 为后卫；某些技能只影响前锋或后卫 |
| 战线天气 | 某些 round 前锋物防 +1，某些 round 后卫魔防 +1 |
| 护送站位 | 盾兵或骑士与英雄相邻时，英雄受到物理伤害 -1，每 round 一次 |
| 狂战扩展 | 余波可根据站位变成“横扫前锋”或“震荡相邻” |

### 1.6 Balatro：有限槽位和可见计算链

Balatro 的启发不是扑克，而是：

- 持续规则修改器必须有槽位限制。
- 算分过程可视化本身就是爽感。
- 很多效果需要围绕条件构筑。

翻译到本项目：

- 遗物、副官、天气、称号都不能无限叠加。
- 预测面板应该像“战斗计算卷轴”，让玩家享受看懂规则链条。
- 强效果必须有条件，不能无脑常驻。

我的机制提案：

| 提案 | 说明 |
|---|---|
| 遗物槽 | 对战成长版每方最多 3 个遗物，满了必须替换或跳过 |
| 副官槽 | 每个英雄 1 个副官槽，少数高级 Class 解锁第二槽 |
| 结算卷轴 | 预测显示：基础伤害 → 技能 → 防御 → 盾 → 守护 → 最终伤害 |
| 条件遗物 | 遗物不直接全体 +1，而是“本回合第三次行动 +1”这类可构筑条件 |

### 1.7 Marvel Snap：地点预告和下注心理

Marvel Snap 的地点随机很适合启发天气：

- 地点逐步揭示，玩家有适应空间。
- 每局地点不同，但规则公开。
- Snap 是风险加倍，不只是赢输。

翻译到本项目：

- 天气必须预告，不要突然改变伤害。
- 天气可以是对战模式的“地点”，让每局有变化。
- 下注机制可以改成 BP 目标，而不是天梯积分。

我的机制提案：

| 提案 | 说明 |
|---|---|
| 天气队列 | 显示当前天气、下一天气、再下一天气的图标 |
| 战术下注 | 玩家可宣告本回合目标，如“破盾”或“击破”，成功得 BP，失败给对手 BP |
| 地形令牌 | 每个 slot 有小地形，如高地/废墟/法阵，影响一次伤害或 BP |

### 1.8 Darkest Dungeon：角色损耗和情感重量

Darkest Dungeon 的价值：

- 角色不是一张资源卡，而是会积累伤痕和故事的人。
- 压力、怪癖、永久死亡让选择有情感代价。
- 营地和治疗是战斗外决策。

翻译到本项目：

- 对战模式不做长期压力，但可以测试“称号/伤痕/荣誉”这种轻量人格化。
- 单机未来可以保留 HP、伤病、士兵死亡。
- 普通兵如果能成长为副官，就会更像“我带出来的人”。

我的机制提案：

| 提案 | 说明 |
|---|---|
| 战斗称号 | 本局内完成特殊行为的单位获得临时称号，如“破阵者”“护旗者” |
| 伤痕奖励 | 未来单机中，英雄可带负面伤痕换强力成长 |
| MVP 记录 | 结算显示造成最多伤害、承受最多伤害、治疗最多、贡献 BP 最多 |
| 老兵回忆 | 普通兵成为副官时，记录其曾经的战斗行为 |

### 1.9 Roguebook / Griftlands：双角色关系和多牌组身份

Roguebook 强调双英雄组合，Griftlands 把战斗和交涉分成不同牌组。对本项目的启发是：

- 构筑不一定只围绕一个主角。
- 两个角色之间的组合关系可以成为 build 核心。
- 同一个角色可在不同系统中承担不同身份。

翻译到本项目：

- 英雄之间可以有“羁绊奖励”，但要谨慎，别让固定组合成为唯一答案。
- 公主/怪物、骑士/公主、占卜师/魔法使、德鲁伊/农民可以形成轻羁绊。

我的机制提案：

| 羁绊 | 触发 |
|---|---|
| 王庭羁绊 | 公主和骑士同时在场，第一次守护后公主获得 1 BP |
| 星火羁绊 | 占卜师和魔法使同时在场，第一次炎上造成伤害后获得星兆 |
| 春林羁绊 | 农民和德鲁伊同时在场，春播被驱散时农民获得 1 BP |
| 美女与野兽 | 公主和怪物同时在场，怪物绝对追击 +1，但公主阵亡后怪物进入暴走 |

### 1.10 MTG / Gwent：关键词、阵营和反制

MTG 和 Gwent 的启发：

- 关键词能长期承载复杂规则。
- 阵营/颜色/职业不是装饰，而是约束可用能力。
- 反制机制必须可见，否则玩家只觉得被阴。

翻译到本项目：

- 英雄、普通兵、遗物、天气都需要统一关键词。
- 物理/魔法、防御/辅助、英雄/士兵应成为规则标签。
- 对战模式中尽量避免隐藏反制，所有危险都应能 hover 或预告。

## 2. 我建议新增的机制语法层

如果未来要承载用户提出的所有系统，当前 GDD 需要多一层“机制语法”，否则会变成很多特例。

### 2.1 单位标签

每张卡可有多个标签：

| 标签 | 用途 |
|---|---|
| `hero` | 可进行英雄升级、转职、装备高级 Class |
| `soldier` | 可升军衔、可成为副官 |
| `physical` | 物理攻击类型 |
| `magical` | 魔法攻击类型 |
| `royal` | 公主、骑士、王旗相关 |
| `arcane` | 魔法使、占卜师、术士相关 |
| `wild` | 怪物、德鲁伊、农民相关 |
| `guard` | 防御/护送相关 |
| `support` | 治疗、驱散、BP 支援相关 |

这些标签可以服务奖励池和遗物，而不是每个遗物都手写角色名单。

### 2.2 资源类型

当前有 AP，未来可以有：

| 资源 | 生命周期 | 用途 |
|---|---|---|
| AP | 单回合 | 行动和防御 |
| BP / Valor | 单局累计，当前上限 10 | 奖励购买、成长消费 |
| Upgrade Charge | 单位个人 | 英雄或士兵成长进度 |
| Relic Slot | 单局容量 | 限制遗物数量 |
| Adjutant Slot | 单位容量 | 限制副官数量 |

### 2.3 成长来源

所有成长都应标记来源：

| 来源 | 例子 |
|---|---|
| 固定奖励窗口 | 三选一 |
| 英雄个人条件 | 公主累计治疗 |
| 普通兵立功 | 士兵击破或保护 |
| 遗物效果 | 战术旗帜 |
| 天气事件 | 战鼓回响 |
| 副官加成 | 盾兵副官 |

这有助于 hover 和日志解释。

## 3. BP / 战功系统：我建议的三个版本

用户提出 BP 是非常关键的。问题是它既可以成为成长引擎，也可能成为滚雪球引擎。

### 3.1 版本 A：简单战功

适合第一版 prototype。

```text
初始 BP：5。
BP 上限：10。
每回合最多获得 3 BP。

+1 BP：每回合低保。
+1 BP：破坏敌方共享盾。
+1 BP：主动攻击造成 HP 伤害。
```

特点：

- 容易理解。
- 低保保证双方都能参与成长系统。
- 主动攻击和破盾获得一点额外节奏。
- 有每回合上限，压制爆炸。
- 不奖励击杀，降低滚雪球风险。

### 3.2 版本 B：角色贡献型 BP

适合后续成长版。

```text
BP 分为全局 BP 和个人贡献。

全局 BP 用于固定奖励窗口消费。
个人贡献提高该单位出现在升级奖励中的概率。
```

例：

- 公主有效治疗给全局 +1 BP，同时增加公主升级权重。
- 盾兵保护英雄给全局 +1 BP，同时增加盾兵军衔奖励权重。
- 魔法使炎上造成伤害不一定立刻给大量 BP，但增加魔法奖励权重。

好处：

- 玩家自然觉得“谁有贡献，谁更容易成长”。
- 奖励随机不再完全无脑。

### 3.3 版本 C：战术目标型 BP

适合高级模式或未来单机。

每 round 给双方一个公开目标：

| 目标 | 成功奖励 |
---|---|
| 本 round 首次破盾 | +2 BP |
| 本 round 用普通兵造成 HP 伤害 | +1 BP |
| 本 round 治疗至少 2 HP | +2 BP |
| 本 round 用魔法击中物防高的单位 | +1 BP |
| 本 round 让一个 HP <= 3 的角色存活到回合结束 | +2 BP |

优点：

- 每 round 都有短期规划。
- 天气和目标可以组合。

风险：

- UI 复杂。
- 新玩家可能不知道先做什么。

### 3.4 我的倾向

对战模式第一版用 A。

成长模式第二版用 B。

单机或高级模式再试 C。

## 4. BP 奖励池：更细的设计

### 4.1 奖励类别

建议把奖励分成 8 类：

| 类别 | 作用 |
---|---|
| Hero Skill | 英雄第一次升级，解锁技能 |
| Hero Stat | 英雄第二次升级，强化属性 |
| Hero Class | 英雄第三次升级，转职 |
| Soldier Rank | 普通兵升军衔 |
| Soldier Recruit | 获得新普通兵或替换普通兵 |
| Adjutant | 副官相关，升满后出现 |
| Relic | 全局持续 modifier |
| Tactical Order | 一次性或短期战术指令 |

### 4.2 奖励出现规则

为了避免完全随机：

- 如果玩家有未解锁技能的英雄，Hero Skill 权重提高。
- 如果玩家没有治疗来源，Support/Soldier Recruit 权重提高。
- 如果玩家普通兵接近满军衔，Adjutant 权重提高。
- 如果玩家落后，防守、恢复、战术指令权重提高。
- 如果玩家领先，强输出遗物权重降低，锦上添花奖励减少。

### 4.3 奖励成本模型

曾经讨论过“累计 BP 推进等级、可用 BP 购买奖励”的双轨模型，但当前结论是太复杂。现在更适合这样理解：

```text
Available BP = 当前可用战功
BP Cap = 10
Reward Window = 固定 round 出现的三选一消费节点
```

这样玩家只有一层资源：

- Available BP 决定当前能买多强奖励，象征可调度资源。
- 跳过奖励是在为后续更高价值选项蓄力。

这比“升级自动免费拿奖励”更适合当前对战测试，也更容易拉表。

### 4.4 当前奖励窗口规则

当前对战 prototype 先固定为：

```text
初始 BP：5。
BP 上限：10。
Round 1：第一次奖励窗口。
之后每隔 2 个 round：再次奖励窗口。
每次窗口：三选一。
每次窗口：1 次 reset。
允许跳过：不花 BP，不拿奖励，保留 BP。
```

这套规则的重点不是“模拟完整肉鸽经济”，而是验证玩家是否会在一场短局内产生清楚的资源计划：

```text
现在买小奖励
还是跳过攒 BP
还是 reset 找更适合当前局面的奖励
```

### 4.5 三选一奖励模板

每个奖励都用统一模板写：

```text
名称：
类别：
BP 成本：
对象：
效果：
持续时间：
可见反馈：
适合谁：
风险：
```

例：

```text
王旗鼓舞
类别：Tactical Order
成本：2 BP
对象：己方一个普通兵
效果：该普通兵本回合攻击 +1；若造成 HP 伤害，获得 1 BP。
持续：本回合
适合：普通兵路线、公主指挥路线
风险：低成本循环，需要每回合一次限制
```

## 10. 物防/魔防的五个版本

用户提的物防/魔防很重要，但固定减伤会很危险。我建议列出多个版本。

### 10.1 版本 A：基础固定防御

- 卡面有物防/魔防。
- 对应类型伤害先减防御。
- 数值 0～2。

优点：直观。  
风险：低攻角色打不动。

### 10.2 版本 B：只防主动攻击

- 物防/魔防只减少主动攻击，不减少反击、技能、状态。

优点：保护 TTK，不压死状态技能。  
风险：玩家要记一条例外。

### 10.3 版本 C：防御有穿透下限

- 防御最多把伤害减到 1，除非盾继续吸收。

优点：低攻角色仍有价值。  
风险：防御参数存在感下降。

### 10.4 版本 D：防御改为护甲标签

不用数字，改关键词：

- 重甲：受到物理主动伤害 -1。
- 抗魔：受到魔法主动伤害 -1。
- 软甲：受到反击伤害 -1。

优点：卡面更简洁。  
风险：不如数字适合成长。

### 10.5 版本 E：防御只来自装备/副官/天气

基础卡不加防御参数。防御是成长系统的一部分。

优点：当前规则不膨胀。  
风险：物理/魔法分类仍暂时不够活跃。

我的倾向：

```text
若下一步要先测物理/魔法分类，做版本 A，但数值上限 1。
若下一步要先测成长系统，做版本 E，把防御留给副官和遗物。
```

## 11. 对战模式实验分支

为了不把正式对战打碎，可以给设计文档定义多个实验模式。

| 模式 | 用途 |
---|---|
| Classic 4 Hero | 当前基线，验证纯英雄平衡 |
| Defense Lab | 只加入物防/魔防 |
| Soldier Mix | 1 英雄 + 3 普通兵 |
| BP Reward | 加 BP 和固定奖励窗口 |
| Hero Awakening | 加英雄第一次升级 |
| Class Trial | 只测试 2 个英雄的三段升级 |
| Adjutant Lab | 测普通兵满级副官 |
| Relic Weather | 测遗物和天气 |

这对 GDD 很重要：对战模式是试验场，但不是每个实验都要进入默认规则。

## 12. 我建议的优先 prototype 组合

### 12.1 第一阶段：降低认知负担

做：

- 1 英雄 + 3 普通兵。
- 普通兵只用剑兵、盾兵、术士、牧师。
- 不加 BP。
- 不加遗物。

验证：

- 玩家是否更容易进入游戏。
- 英雄是否更特别。
- 普通兵是否无聊。

### 12.2 第二阶段：BP 与固定奖励窗口

做：

- 简单 BP 版本 A。
- 固定 round 三选一奖励窗口。
- 奖励只有小属性、普通兵 Rank、英雄第一次技能解锁。

验证：

- BP 是否让玩家改变行动。
- 是否滚雪球。
- 三选一是否拖节奏。

### 12.3 第三阶段：英雄第一次升级

做：

- 每个英雄只做两个初级技能。
- 先只做公主、农民、魔法使、骑士四名。
- 不做转职。

验证：

- 白卡英雄到技能解锁是否有快感。
- 玩家是否记得自己选的技能。

### 12.4 第四阶段：物防/魔防

做：

- 只上限 1。
- 明确预测显示。
- 不和天气/遗物一起测。

验证：

- 伤害类型是否更有意义。
- 低攻角色是否失效。

### 12.5 第五阶段：副官和遗物

做：

- Rank 2 普通兵可副官。
- 副官只提供简单属性。
- 每方最多 2 遗物。

验证：

- 玩家是否愿意把士兵从战场转成英雄强化。
- 槽位限制是否足够。

### 12.6 第六阶段：天气

做：

- 只 4 种天气。
- 当前 + 下一天气预告。
- 每局最多出现 3 次天气。

验证：

- 玩家是否会为天气调整计划。
- 是否觉得不公平。

## 13. 当前 GDD 应该新增的内容

### 13.1 对战模式定位

建议 GDD 明确写：

> 对战模式是局内战斗和未来单机机制的实验场。它优先验证战斗选择、成长奖励、UI 可读性和短局节奏，而不是完整复刻最终单机爬塔。

### 13.2 新支柱候选

上一份已提出“短局中也能成长”。我建议再加一个更具体的：

> **成长必须被看见，也必须能被反制。**  
> 英雄升级、普通兵军衔、副官、遗物和天气都要公开显示，进入预测，并给对手至少一个行动窗口。

### 13.3 系统进入标准

任何新系统进 GDD 前问：

| 问题 | 目标 |
---|---|
| 它让玩家多了什么动词 | 选择、升级、附着、预告、下注 |
| 它支持哪条支柱 | 不能只是酷 |
| 它是否创造取舍 | 有成本或机会成本 |
| 它是否有反制 | 对手能观察和应对 |
| 它是否加重认知 | 新玩家在哪里学会 |
| 它是否能预测 | 伤害和 Cost 是否能显示 |
| 它是否会滚雪球 | 是否有平衡循环 |
| 它是否需要新 UI | hover、日志、图标、音效 |

## 14. 方向性结论

我认为项目下一阶段最有生命力的方向不是“给现有 8 英雄继续堆技能”，而是：

```text
英雄从白卡开始
普通兵承担低复杂度战斗
BP/固定奖励窗口提供局内成长节奏
英雄升级提供 JRPG 角色养成
普通兵军衔和副官把士兵转化为英雄成长材料
遗物和天气提供 build 与战场变化
对战模式分实验分支逐个验证这些系统
```

最应该先做的不是转职和遗物海量内容，而是一个小而完整的成长 prototype：

```text
1 英雄 + 3 普通兵
简单 BP
固定 round 奖励窗口
英雄第一次升级二选一
普通兵 Rank 0～2
```

如果这个小循环能成立，后面的转职、副官、遗物、天气都会有根。

如果这个小循环不成立，直接做 8 英雄 × 2 高级 Class 会很容易变成漂亮但失控的大工程。
