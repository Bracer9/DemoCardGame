# 共有盾结算修正记录（2026-06-30）

## 当前规则

共有盾不是角色身上的减伤 Buff，而是横在队伍前方的共享防线。它没有独立可点击实体，但在数值上拥有自己的防御属性。

默认值：

- 共有盾值：由防御阵形产生。
- 共有盾物防：0。
- 共有盾魔防：0。

## 伤害结算顺序

```text
原始伤害
→ 攻击方增伤/减伤
→ 共有盾物防/魔防
→ 共有盾吸收
→ 穿透剩余伤害
→ 目标角色物防/魔防
→ 角色侧减伤/预见/守护等后置保护
→ 扣 HP
```

关键原则：

- 共有盾先于目标角色防御结算。
- 只有穿透共有盾的剩余伤害，才会受到被攻击角色自身物防/魔防影响。
- 如果共有盾已经完全吸收主动攻击伤害，骑士守护不会发动。
- 未来 Role Action、遗物、天气或职业强化可以修改共有盾的物防/魔防。

## 实装位置

- `Domain/GameState.cs`：`PlayerState` 增加共有盾物防/魔防字段。
- `Domain/GameEngine.cs`：实战伤害管线改为共有盾先结算。
- `Api/AttackPreviewService.cs`：预测伤害使用相同顺序。
- `Api/GameDtos.cs`：玩家 DTO 暴露共有盾物防/魔防。
- `wwwroot/app.js`：防御阵形 hover HUD 显示盾值、物防、魔防。
